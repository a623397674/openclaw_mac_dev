# 🏢 一人公司 - 单 Bot 多角色方案

## 问题
QQ 群无法邀请多个 Bot 账号（QQ 开放平台限制）

## ✅ 解决方案：单 Bot 多角色模式

### 核心思路
用 **客服小暖** (QQ: 1903488919) 作为统一入口，在群聊中自动模拟所有角色的观点。

### 配置步骤

#### 1. 创建普通 QQ 群
- 群名称：**一人公司 🏢**
- 邀请 **客服小暖** (QQ: 1903488919) 进群
- 你和其他人类成员进群

#### 2. 客服小暖已配置多角色回复

客服小暖的 `SOUL.md` 已更新，检测到群聊消息时会自动：

1. 以客服身份开场
2. 模拟 4 个角色的专业意见（产品/后端/前端/财务）
3. 以客服身份结尾，询问是否需要深入讨论

### 使用示例

**你在群里发送：**
```
我们要做一个新的用户系统
```

**客服小暖回复：**
```
🎧 客服小暖：收到！这个问题我来帮大家问问~

---
📊 产品小助手：建议先明确目标用户和使用场景，核心功能是注册登录还是包含会员体系？
⚙️ 后端老张：数据库用 PostgreSQL 吧，用户表设计要考虑扩展性，我来搞表结构。
🎨 前端小美：前端用 React + TypeScript，登录注册页面要适配移动端哦。
💰 财务小慧：这个项目的预算多少？开发成本记得记录一下~
---

🎧 客服小暖：以上是各部门的初步意见，需要深入讨论哪个方向吗？
```

## 📱 其他渠道方案

### 方案 B：飞书群（推荐用于工作）

飞书支持群聊机器人，可以配置多 Agent：

1. 创建飞书群
2. 添加飞书机器人
3. 配置 OpenClaw 飞书渠道
4. 使用 broadcast 功能实现多 Agent 回复

**优势：** 企业级功能，支持卡片消息、审批流等

### 方案 C：私聊 + 话题标签

用私聊 Bot，通过标签指定角色：

```
#产品 帮我写个需求文档
#后端 设计一个数据库表
#前端 写个登录页面
#财务 记录一笔支出
```

客服小暖根据标签调用不同角色的系统提示。

### 方案 D：Discord/Telegram

这些平台支持多 Bot 进群：

- Discord：可以邀请多个 Bot 到服务器
- Telegram：支持多个 Bot 到群组
- 配置 OpenClaw 对应渠道即可

## 🔧 技术实现

### 群聊检测

客服小暖通过消息类型判断：

```javascript
// 伪代码
if (message.chatType === 'group') {
  // 启用多角色回复模式
  return generateMultiRoleResponse(message);
} else {
  // 普通客服回复
  return normalSupportResponse(message);
}
```

### 多角色回复生成

使用系统提示词引导模型模拟多角色：

```markdown
你是一个一人公司的客服，群里有 5 个角色：
- 产品小助手（产品视角）
- 后端老张（技术视角）
- 前端小美（UI/UX 视角）
- 财务小慧（财务视角）
- 客服小暖（你，协调沟通）

当用户在群里发言时，依次模拟每个角色的简短意见。
```

## 📊 Token 消耗

多角色模式会消耗更多 Token（约 2-3 倍），因为：
- 需要生成更长的回复
- 每个角色视角都需要上下文

财务小慧的日报会统计实际消耗。
